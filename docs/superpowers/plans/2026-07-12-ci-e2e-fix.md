# CI E2E Fix — Missing Playwright System Dependencies

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Fix 16 failing e2e tests in CI caused by missing Playwright system dependencies (`libcups2`, `libpango-1.0-0`, `libcairo2`), and revert unnecessary `app.js` changes from commit `ddd8201`.

**Architecture:** The CI container (`debian:trixie-slim`) has all Chromium deps installed except the three libraries Playwright's headless shell needs for text layout, graphics, and printing. Without them, HTML constraint validation (`checkValidity()`, `:invalid` pseudo-class) silently fails, breaking all tests that depend on form interaction. The fix adds the missing apt packages and reverts the dead-weight `app.js` change that was a misdirected attempt at a fix.

**Tech Stack:** GitHub Actions CI, Debian Trixie, Playwright, apt-get

---

## Root Cause Summary

Playwright warns on install:
```
Host system is missing dependencies to run browsers.
apt-get install libcups2 libpango-1.0-0 libcairo2
```

These three packages are absent from CI's apt-get install list. The `test-deps/setup.sh` script downloads exactly these libraries for local development, confirming they are required. Without them, Chromium Headless Shell starts but constraint validation APIs return incorrect results, causing 16/47 e2e tests to fail (all that depend on `checkValidity()` or the `:invalid` pseudo-class).

### Why this was missed

The `playwright.config.js` `LD_LIBRARY_PATH` handling was intended to supply these libs from `test-deps/lib/`, but that directory only exists locally (created by `test-deps/setup.sh`, which is never run in CI). In CI, `existsSync(testDepsLib)` is `false`, so the path is never set.

### What stays vs. what goes

Commit `ddd8201` changed both `app.js` and `playwright.config.js`. Only the `app.js` change is reverted (Task 3). The `playwright.config.js` change is **kept** — it fixed a real bug where `launchOptions.env` was overridden by `...process.env`, making the old code ineffective even locally. In CI it's a no-op, but locally it correctly supplies bundled libs.

Commits `248c0f7` and `08df428` are **kept as-is**. Their test improvements (selector waits, missing `blur()` calls, accordion-open guards) are net-positive. The remaining gaps they missed are filled by Task 2.

### Task 3: Revert unnecessary app.js change from ddd8201

**Files:**
- Modify: `app.js:684-693`

Commit `ddd8201` changed the `U()` function to open all `<details>` before validation, then re-close valid-only sections. This was an attempt to fix validation in CI but `checkValidity()` works regardless of details open state — the change adds runtime code with no effect. Revert to the pre-`ddd8201` logic.

- [ ] **Step 1: Revert the U() function to its pre-ddd8201 state**

In `app.js`, change lines 684-693 from:

```js
    var details = configForm.querySelectorAll('details');
    for (var di = 0; di < details.length; di++) {
      details[di].open = true;
    }
    var fields = configForm.querySelectorAll('input, select, textarea');
    for (var fi = 0; fi < fields.length; fi++) {
      if (fields[fi].name) fields[fi].setAttribute('aria-invalid', fields[fi].checkValidity() ? 'false' : 'true');
    }
    for (var di = 0; di < details.length; di++) {
      if (!details[di].querySelector('[aria-invalid="true"]')) details[di].open = false;
    }
```

To (original logic — validate first, then open sections with invalid fields):

```js
    var fields = configForm.querySelectorAll('input, select, textarea');
    for (var fi = 0; fi < fields.length; fi++) {
      if (fields[fi].name) fields[fi].setAttribute('aria-invalid', fields[fi].checkValidity() ? 'false' : 'true');
    }
    var details = configForm.querySelectorAll('details');
    for (var di = 0; di < details.length; di++) {
      if (details[di].querySelector('[aria-invalid="true"]')) details[di].open = true;
    }
```
